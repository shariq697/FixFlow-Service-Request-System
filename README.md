🚀 FixFlow – Service Request Management System

FixFlow is a relational database system designed to efficiently manage and track service requests within a centralized platform. The project focuses on structured data storage, optimized SQL queries, and reliable data management using relational database principles. It simulates a real-world service request tracking environment with scalable database architecture and efficient record management.

📌 Key Features

- Create, read, update, and delete CRUD service request records
- Fully normalized database schema to reduce redundancy and improve data integrity
- Strong relationships between tables using primary and foreign keys
- Optimized SQL queries for fast and efficient data retrieval
- Structured ticket, user, technician, and comment management
- Scalable database design suitable for real-world service systems

🛠️ Technologies Used

- SQL
- Relational Database Design
- MySQL 
- Python
- HTML

🧱 Database Design Highlights

- Tables normalized using 1NF, 2NF, and 3NF principles
- Well-defined primary and foreign key relationships
- Efficient schema design for querying and reporting
- Structured entity relationships for users, tickets, technicians, and comments
- Organized database architecture focused on scalability and maintainability
